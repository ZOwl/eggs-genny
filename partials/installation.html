Installation/Setup

It's pretty easy to install Yeoman and then eggs-genny, which are both available on the npm registry. Dependencies are installed using Bower, and if you aren't familiar with these it's totally fine because eggs-genny takes care of all that for you. It uses Gulp as a task runner (Grunt and pure-npm task-runners coming soon!), and even if you don't use it you should be able to follow the usage guide/wiki and understand what's going on.

Step One:

Install Yeoman. That `-g` means gangster --er, I mean 'global'-- as in it is globally available on your system.

npm install -g yo

As we said, you'll need Bower & Gulp, so if you want you can install as many as you want at the same time:

npm install -g yo bower gulp

Problems? See below.

Step Two:

Install the greatest Yeoman genny known to man:

npm install -g generator-eggs-genny

Thats it. You've got eggs-genny on your system.

Step Three:

cd into you rag-tag little project and run:

yo eggs-genny

This is where shit gets real.

Step Four:

Answer some questions about you and your app. Then answer some questions about the dependencies you want.

You can learn all about the dependencies that eggs-genny has to offer on the dependency pages. If you are using the GSAP animation platform, you will also be asked additional questions about which plugins you would like to use.

Step Five (some users):

So if you saw a bunch of like errors or red stuff toward the end of that, then you may need to do this one tiny extra step. eggs-genny attempts to do an npm install of the necessary dependencies. Depending on how permissions are configured on your system, you may also need to run it with sudo after doing step 4. You can see more about this below.

What the shit is this?

Don't panic, it's all good. All you need to do is throw sudo in front of those commands and you should be good. Like this:

sudo npm install

And then type your password. You probably won't be able to see what you're typing DON'T SCREW UP! Just kidding it's nbd if you do, you'll just have to re-type it.

sudo is short for pseudoephedrine, which is what some commands need to be run on some people's systems. Not actually though.
