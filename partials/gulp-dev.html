Gulp - Development

Even if you have no experience with a task runner, that's fine because I'm going to show you the exact commands you have to copy & paste into your terminal to start developing like a boss. Here's a rundown of the essential development tasks that eggs-genny offers (we'll cover tasks for building a production-ready app over here)

Default

gulp

In Gulp, the gulp task is the default task. It sets up a development environment, by running some tasks outlined below:

serve-me

This sets up a localhost server. The port it listens to by default is :6969, because I am inappropriate. So you can throw this url into your browser to see your app running right away:

http://localhost:6969/app/index.html

reload-me

Okay so if you aren't aware yet of the greatest developer tool ever invented, feast your eyes on LiveReload which is a free Chrome extension that, when configured, runs with Gulp. So you have the LiveReload going on that url above, and then whenever your files are changed, the page automagically reloads.

Here's what you do. You download the Chrome extension and run gulp (or gulp reload-me), then in the browser window running your app turn on LiveReload. When Gulp spots changes to your main files under app/, such as your css/style.less file or your js/script.js file, or even your .hmtl files, it will trigger the LiveReload and the page is reloaded for you!

As mentioned, Gulp watches some files and runs tasks depending on what files are changed.

  • all .less files are compiles to css on save
  • any .js files changed get run through js-hint and reported to the console
  • and then as stated if any changes to main files are detected then the app will reload

validate-me

gulp validate-me

This one isn't part of the default gulp task or the `reload-me` task (though the validate-js task is used when .js files are saved). You can run this at any time to test your js and css at the same time. By the way, the command to just test the css is validate-css.

Utilities

These are just a few functions that aid in the task running process, such as timestamping the reload and logging things in a visible way. There is also a custom reporter for css-lint.

Supported Browsers

There is a variable at the top of the Gulpfile called supportedBrowsers which is how you can set which browsers Autoprefixer will add vendor prefixes to support. By default it goes back to like the stone age:

supportedBrowsers = [ 'last 4 versions', '> 0.5%', 'ie 7', 'ff 3', 'Firefox ESR', 'Android 2.1' ];

...which of course you can reset to whatever you like. Read more on Autoprefixer and the various browser strings over here.
